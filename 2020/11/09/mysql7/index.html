MySQL语句

  • ALTER TABLE 更新已存在的表的模式
  • COMMIT将事务处理写到数据表
  • CREATE INDEX用于在一个或多个列上创建索引
  • CREATE PROCEDURE创建存储过程
  • CREATE TABLE创建新数据表库
  • CREATE USER添加新的用户账号
  • CREATE VIEW创建一个或多个 表上的新视图
  • DELETE从 表中删除一行或多行
  • DROP永久的删除数据库对象（表，视图，索引）
  • INSERT给表增加一行
  • INSERT SELECT插入SELECT的结果到一个表
  • ROLLBACK撤销一个事务处理块
  • SAVEPOINT为使用ROLLBACK语句设置保留点
  • SELECT从一个或多个表（视图）检索数据
  • START TRANSACTION表示一个新的事物处理快开始
  • UPDATE更新表中的一行或多行
  • ♥

MYSQL数据类型

数据类型是定义列中可以存储什么数据以及该数据实际怎么存储的基本规则

  • 数据类型允许限制可存储在列中的数据（数值数据类型只能接收数值
  • 数据类型允许在内部更有效的存储数据（用比文本串更简洁的格式存储数值和日期时间值
  • 数据类型允许变换排序顺序（串处理1在10前面，10又在2 前，即字典序，而数值数据类型，数值才可正确排序）

1.串数据类型

定长穿：长度在创建表时指定，CHAR属于定长串

变长串可存储可变长文本 （有些变长数据类型具有最大定长，有些则是完全变长）TEXT属于变长串类型

既然变长数据类型这样灵活，为什么还要使用定长数据类型？

回答是因为性能。MySQL处理定长列远比处理变长列快得多。此外，MySQL不 允许对变长列（或一个列的可变部分）进行索引。这也会极大地影响性能

2.数值数据类型

有符号或无符号 所有数值数据类型（除BIT和BOOLEAN外） 都可以有符号或无符号。有符号数值列可以存储正或负的数 值，无符号数值列只能存储正数。默认情况为有符号，但如 果你知道自己不需要存储负值，可以使用UNSIGNED关键字， 这样做将允许你存储两倍大小的值。

存储货币数据类型 MySQL中没有专门存储货币的数据类 型，一般情况下使用DECIMAL(8, 2)

MySQL保留字

MySQL是由关键字组成的语言，关键字是一些用于执行MySQL操作
的特殊词汇。在命名数据库、表、列和其他数据库对象时，一定不要使 用这些关键字。因此，这些关键字是一定要保留的
